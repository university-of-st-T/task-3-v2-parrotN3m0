# task_3_v2

**Задача 3. Калькулятор стоимости заказа с гибкими скидками**  

### 📁 Как называется файл с решением?
Файл с кодом должен называться **`calculate_total.py`** (и лежать в корне репозитория). Именно такой файл будут искать автотесты.

### 📝 Что нужно написать в файле?
Создать файл `calculate_total.py` и написать в нём функцию **`calculate_total`** точно с такими параметрами, как в условии:

```python
def calculate_total(prices, *discounts, **options):

```

### Как должна работать функция?
- **prices** — список цен товаров (например, `[100, 200, 300]`).  
- **\*discounts** — скидки в процентах. Их может быть любое количество (например, `10, 5`). Они применяются **последовательно**: сначала одна скидка, потом другая.  
- **\*\*options** — дополнительные настройки:  
  - `tax` — налог в процентах (по умолчанию 0).  
  - `round_to` — до скольких знаков округлять результат. Если `None` — не округлять. По умолчанию 2.

**Формула расчёта:**
1. Суммируем цены из `prices`.
2. Для каждой скидки умножаем сумму на `(1 - скидка/100)`.
3. Добавляем налог: умножаем на `(1 + налог/100)`.
4. Если `round_to` не равен `None`, округляем результат функцией `round(результат, round_to)`.
5. Возвращаем итоговое число (тип `float`).

### 🧪 Пример из условия
```python
prices = [100, 200, 300]
total = calculate_total(prices, 10, 5, tax=20, round_to=1)
print(total)  # Должно напечатать 615.6
```
Как это получается:
- Сумма = 600
- После скидки 10%: 600 × 0.9 = 540
- После скидки 5%: 540 × 0.95 = 513
- Налог 20%: 513 × 1.2 = 615.6
- Округление до 1 знака → 615.6

### ✅ Что важно для автотестов
1. **Имя файла обязательно:** `calculate_total.py`.
2. **Имя функции обязательно:** `calculate_total`.
3. **Параметры должны быть именно такими** — порядок и имена важны.
4. **Не использовать `print()` внутри функции** — лишний вывод может помешать проверке. (Можно добавлять временно для отладки, но перед отправкой убрать.)
5. **Возвращать результат, а не печатать его**.
